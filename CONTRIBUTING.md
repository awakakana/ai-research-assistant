# CONTRIBUTING

## 贡献指南

感谢你对本项目的关注！欢迎任何形式的贡献。


### 贡献类型

#### 1. 新增角色Prompt

如果你设计了一个新的研究辅助角色，请确保：

- 遵循现有角色的格式（头部信息、使用说明、角色定位、核心任务、输出格式、注意事项、版本历史）
- 包含语言选择步骤
- 提供使用说明和适用场景
- 标注推荐使用的AI及参数

提交方式：
- 将角色Prompt文件放入对应阶段的文件夹
- 提交Pull Request

#### 2. 优化现有角色

如果你发现现有角色有改进空间：
- 语言表达不清晰
- 输出格式可以更好
- 遗漏了重要检查项

请提交Issue或Pull Request说明改进建议。

#### 3. 新增领域知识速成包

如果你熟悉某个领域（如神经科学、免疫学、合成生物学等），欢迎贡献领域知识包：

- 核心概念（15-20个术语）
- 常用模型/系统
- 常用技术方法
- 入门文献推荐（综述+标志性研究）
- 工具与资源

模板参考：`资源库/领域知识/模板.md`

#### 4. 新增工作流

如果你有常用的角色组合顺序，欢迎贡献预定义工作流：

- 工作流名称
- 适用场景
- 角色使用顺序
- 各阶段输入/输出说明

#### 5. 示例对话

如果你有使用本工具集的实际对话示例，欢迎贡献到`示例/`目录：

- 用户输入
- AI输出
- 使用心得


### 提交规范

#### 文件命名

- 角色Prompt：使用中文名称，如`点子孵化器.md`
- 引导师：统一命名为`引导师.md`
- 领域知识包：使用领域名称，如`神经科学.md`

#### 版本号

新贡献的角色版本号从`v0.0.1`开始。

#### Pull Request

1. Fork本仓库
2. 创建你的特性分支（`git checkout -b feature/AmazingFeature`）
3. 提交你的更改（`git commit -m 'Add some AmazingFeature'`）
4. 推送到分支（`git push origin feature/AmazingFeature`）
5. 提交Pull Request


### 行为准则

- 尊重他人，友善交流
- 贡献内容需原创或已获授权
- 描述清晰，便于他人理解
