# AI研究助手（ai-research-assistant）

> 一套开源的、以Prompt为核心的AI辅助研究工作流

**让研究者专注于思考，让AI处理流程。**


## 项目简介

本项目是一套结构化的角色Prompt工具集，将AI转化为可调用的“研究专家”——设计审查官、缺漏分析师、论文结构师等。用户无需学习提示词工程，只需**复制Prompt到AI对话窗口**，即可获得结构化输出。

**零门槛**：无需安装、注册或付费，任何AI对话工具都能用。

**18个角色**：覆盖从模糊想法到答辩演练的全流程。

**开源可扩展**：用户可自建角色，可贡献知识库。


## 快速开始

### 第一步：选择起始阶段

| 你的情况 | 起始阶段 | 操作 |
|---------|---------|------|
| 第一次使用，不知道从哪开始 | **阶段0** | 使用[全局引导师](阶段0_全局引导/全局引导师.md) |
| 有模糊想法，不知道怎么设计实验 | **阶段1** | 使用[点子孵化器](阶段1_创意孵化/点子孵化器.md) |
| 有实验方案，需要审查优化 | **阶段2** | 使用[设计审查官](阶段2_研究执行/设计审查官.md) |
| 有数据，准备写论文 | **阶段2** | 使用[论文结构师](阶段2_研究执行/论文结构师.md) |
| 准备答辩/组会 | **阶段3** | 使用[组会模拟员](阶段3_学术表达/组会模拟员.md) |
| 有A领域经验，想转到B领域 | **阶段x** | 使用[跨域咨询师](阶段x_跨域接入/跨域咨询师.md) |

### 第二步：复制Prompt

点击上方角色链接，复制Prompt内容。

### 第三步：粘贴到AI对话

粘贴到任意AI对话工具（推荐DeepSeek或Qwen，免费），AI会首先询问语言偏好，然后按格式输出分析结果。


## 多语言支持

首次使用任意角色时，AI会询问你的语言偏好。当前支持：中文、English。


## 项目结构

```
ai-research-assistant/
├── 阶段0_全局引导/        # 入口：全局引导师
├── 阶段1_创意孵化/        # 从模糊想法到实验方案
├── 阶段2_研究执行/        # 实验设计、数据、论文
├── 阶段3_学术表达/        # 组会、答辩模拟
├── 阶段x_跨域接入/        # 跨领域研究支持
├── 角色设计器/            # 自定义新角色
├── 资源库/                # 工具推荐、领域知识包
├── 工作流/                # 预定义研究路径
└── 示例/                  # 示例对话
```


## 角色清单

| 阶段 | 角色 | 功能 |
|------|------|------|
| 阶段0 | 全局引导师 | 诊断用户状态，推荐起始阶段和角色 |
| 阶段1 | 点子孵化器 | 模糊想法→初步框架 |
| | 可行性分析师 | 评估技术、时间、经费 |
| | 创新性评估师 | 判断新意，指导文献检索 |
| | 路径规划师 | 输出从想法到实验的路线图 |
| 阶段2 | 设计审查官 | 审查对照组、样本量、统计 |
| | 缺漏分析师 | 识别缺漏、风险、优先级 |
| | 文献助手 | 提取论文核心信息 |
| | 数据化验收员 | 检查数据完整性、异常值 |
| | 论文结构师 | 生成论文大纲、图表建议 |
| 阶段3 | 组会模拟员 | 多位专家模拟提问 |
| 阶段x | 跨域咨询师 | 能力迁移分析、学习路线图 |
| | 领域知识速成师 | 核心概念、经典文献 |
| | 迁移设计顾问 | 实验设计改编 |
| | 文献迁移工具 | 文献范式迁移 |
| 角色设计器 | 角色设计引导师 | 通过对话设计新角色 |
| | 空白角色模板 | 标准化角色Prompt模板 |
